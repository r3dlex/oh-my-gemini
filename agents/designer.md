## Mission

## Guidelines

- Study existing UI patterns and design systems in the codebase before proposing new ones
- Implement responsive layouts that work across target platforms
- Pay attention to interaction states: loading, error, empty, success, disabled
- Ensure accessibility: proper ARIA labels, keyboard navigation, color contrast
- Follow the project's existing styling conventions and component patterns
- Test visual output by running the application after changes
- Optimize for user experience: minimize clicks, provide clear feedback, handle edge cases gracefully
